프로그래머스 - 기둥과 보 설치

18 Sep 2021 Algorithm C++ Online judge 구현/시뮬레이션 Programmers



Problem

2020 KAKAO BLIND RECRUITMENT 기둥과 보 설치

문제는 2차원 공간에 구조물 (기둥 or 보)를 설치/삭제 한 후 최종 구조물의 상태를 return 하는 것이다. 구조물의 정보(x 좌표, y좌표, 구조물 종류, 설치/삭제)들을 담은 2차원 배열이 입력되고, 최종 구조물의 상태(x 좌표, y 좌표, 구조물 종류)를 반환하면 된다. 단, 문제에 나온 특정 조건을 만족해야만 주어진 위치에 기둥과 보를 설치/삭제할 수 있다는 점에 유의해야 한다.


Solution

보와 기둥의 존재 여부를 나타내는 2차원 배열을 각각 만들고, 현재 특정 위치에 구조물이 존재하는 지를 표시한다. 구현할 때, 구조물 설치/삭제 입력에 대해 설치와 삭제 시에 만족해야 하는 특정 조건을 검사하는 것이 까다롭다. 기둥과 보가 존재하기 위해서는 각각의 조건을 만족해야 한다.

  • 기둥
    • 바닥 위에 존재
    • 왼쪽/오른쪽 보의 끝 위에 존재
    • 다른 기둥 위에 존재
  • 보
    • 왼쪽/오른쪽 끝 부분이 기둥 위에 존재
    • 양쪽 끝 부분이 다른 보와 동시에 연결

설치 시에는 위의 조건을 만족하는 지 확인하고, 만족할 경우 설치를 해주면 된다. 반면 삭제할 때는 삭제하려는 기둥, 보의 존재 여부에 영향을 받는 기둥, 보에 대해 위의 유효 조건을 검사해야 한다. 조건을 만족하지 않을 경우 삭제를 할 수 없으므로, 구조물 삭제를 취소해야 한다. x,y 좌표에 있는 기둥 또는 보 삭제 시 영향을 받는 구조물은 다음과 같다.

  • 기둥 (x, y)
    • 바로 위에 존재하는 기둥 (x, y+1)
    • 오른쪽 끝이 기둥 위에 존재하는 보 (x-1, y+1)
    • 왼쪽 끝이 기둥 위에 존재하는 보 (x, y+1)
  • 보 (x, y)
    • 왼쪽 끝 위에 존재하는 기둥 (x, y)
    • 오른쪽 끝 위에 존재하는 기둥 (x+1, y)
    • 왼쪽 끝에 닿아있는 보 (x-1, y)
    • 오른쪽 끝에 닿아있는 보 (x+1, y)

해당 위치에 있는 특정 기둥과 보만을 검사하면, 모든 구조물을 검사하지 않고 삭제가 가능한지 효과적으로 판단할 수 있다. 주의해야 할 점은, 검사해야 하는 위치에 구조물이 있을 경우에만 유효성 검사를 해야 하는 것이다. 이점을 신경쓰지 못해서 모든 케이스를 통과하는 데 많은 시간이 걸렸다.


Code

#include <vector>
using namespace std;

bool map[2][101][101];

// 기둥 놓을 수 있는지 확인
bool check_0(int n, int x, int y){
    if ((y == 0) ||
        (x > 0 && map[1][y][x-1]) ||
        (map[1][y][x]) ||
        (y > 0 && map[0][y-1][x])) {return true;}
    else {return false;}
}

// 보 놓을 수 있는지 확인
bool check_1(int n, int x, int y){
    if ((y > 0 && map[0][y-1][x]) ||
        (x < n && y > 0 && map[0][y-1][x+1]) ||
        (x > 0 && x < n && map[1][y][x-1] && map[1][y][x+1])){return true;}
    else {return false;}
}

// 기둥 지울 수 있는지 확인
bool check_d_0 (int n, int x, int y){
    if ((!map[0][y+1][x] || check_0(n, x, y + 1)) &&
        (!map[1][y+1][x] || check_1(n, x, y + 1)) &&
        (!map[1][y+1][x-1] || check_1(n, x-1, y + 1))){
        return true;
    }else{
        return false;
    }
}

// 보 지울 수 있는지 확인 
bool check_d_1(int n, int x, int y){
    if ((!map[1][y][x-1] || check_1(n, x - 1, y)) &&
        (!map[1][y][x+1] || check_1(n, x + 1, y)) &&
        (!map[0][y][x] || check_0(n, x, y)) &&
        (!map[0][y][x+1] || check_0(n, x + 1, y))){
        return true;
    }else {
        return false;
    }
}

void construct(int n, int x, int y, int a){
    if (a == 0 && check_0(n, x, y)){
        map[0][y][x] = true;
    }else if(a == 1 && check_1(n, x, y)) {
        map[1][y][x] = true;
    }
}

void deconstruct(int n, int x, int y, int a){
    if (a == 0){
        map[0][y][x] = false;
        if (!check_d_0(n, x ,y)) map[0][y][x] = true;

    }else if (a == 1){
        map[1][y][x] = false;
        if (!check_d_1(n, x, y)) map[1][y][x] = true;
    }
}

vector<vector<int>> solution(int n, vector<vector<int>> build_frame) {
    vector<vector<int>> answer;
    
    for (int i = 0; i < build_frame.size(); i++){
        int x = build_frame[i][0];
        int y = build_frame[i][1];
        int a = build_frame[i][2];
        int b = build_frame[i][3];
        
        if (b == 1){
            construct(n, x, y, a);
        }else {
            deconstruct(n, x, y, a);
        }
    }
    
    for (int i = 0; i <= n; i++){
        for (int j = 0; j <= n; j++){
            vector<int> each_0 (3);
            vector<int> each_1 (3);
            
            if (map[0][j][i]) {
                each_0[0] = i;
                each_0[1] = j;
                each_0[2] = 0;
                answer.push_back(each_0);
            }
            
            if (map[1][j][i]){
                each_1[0] = i;
                each_1[1] = j;
                each_1[2] = 1;
                answer.push_back(each_1);
            }
        }
    }
    
    return answer;
}



백준(10816) - 숫자 카드2

16 Sep 2021 Algorithm C++ Online judge Binary search Baekjoon



Problem

Figure 1. 백준 10816 숫자 카드2 (source)


Solution

문제는 N개의 정수 입력이 주어졌을 때, M개의 특정 숫자가 나온 빈도 수를 출력하는 것이다. 숫자의 범위가 (-10,000,000 <= x <= 10,000,000) 밖에 안되기 때문에, 가장 간단한 풀이는 x에 최소값을 더해 양수로 변환한 후 배열에 각 숫자가 발생한 빈도를 count하는 것이다. 다른 방법은 binary search를 응용하는 방법인데, 입력 배열을 오름차순으로 정렬한 다음 아래와 같이 특정 숫자가 나타나는 범위(lower bound, upper bound)를 구해서 푸는 방법이다. 첫번째 방법은 숫자의 범위가 수십억 등 더 큰 단위로 커질 경우 메모리를 많이 사용해야 되며, 대부분의 사람들이 스스로 잘 풀 것이라고 생각한다. 따라서, 개인적으로 정리도 하는 겸 두번째 방법으로 푸는 방법에 대해 소개하려고 한다.


Lower bound

Lower bound는 특정 값(key)보다 크거나 같은 값의 첫번째 위치를 나타낸다. lower bound 값을 찾기 위해서 binary search와 같이 탐색 공간을 반복적으로 절반씩 줄여 나간다. 이때 주의해야 할 것은 탐색 공간을 줄일 때 right와 left의 값을 어떻게 설정하는 지이다. key 값보다 mid 위치의 값이 크거나 같은 경우 lower bound는 mid의 왼편에 존재하게 되는데, 이때 mid가 lower bound가 될 수 있기 때문에, right = mid로 설정해준다. 반면에 key 값이 mid 위치의 값 보다 큰 경우, mid가 lower bound가 될 가능성이 없기 때문에 left = mid + 1으로 설정한다. 이러한 과정을 lower < right 조건이 만족하는 경우에 한해 반복하고, loop가 끝나면 left 값을 반환하게 된다.


Upper bound

Upper bound는 특정 값(key)보다 큰 값이 처음으로 나오는 위치를 나타낸다. 만약 key 값이 mid 위치의 값보다 작은 경우 mid가 upper bound가 될 수 있으므로, right = mid로 설정한다. 반면에 key 값이 mid 위치 값보다 크거나 같은 경우, mid가 upper bound가 될 수 없기 때문에 left를 mid + 1으로 설정한다.


Code

#include <iostream>
#include <algorithm>
using namespace std;

int arr[500001];
int N, M;

int upperbound(int key) {
	int left = 0;
	int right = N;
	int mid;
	
	while (left < right) {
		mid = (left + right) / 2;
		if (key < arr[mid]) right = mid;
		else left = mid + 1;
	}

	return right;
}

int lowerbound(int key) {
	int left = 0;
	int right = N;
	int mid;
	
	while (left < right) {
		mid = (left + right) / 2;
		if (key <= arr[mid]) right = mid;
		else left = mid + 1;
	}

	return left;
}

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(NULL);
	cout.tie(NULL);

	cin >> N;

	for (int i = 0; i < N; i++) {
		cin >> arr[i];
	}

	sort(arr, arr + N);

	cin >> M;

	for (int i = 0; i < M; i++) {
		int num;
		cin >> num;
		
		int lower_idx = lowerbound(num);
		int upper_idx = upperbound(num);

		cout << upper_idx - lower_idx << " ";
	}

	return 0;
}


Time complexity

Sorting과 2M번의 binary search 시간을 더하여 $O( (N+M)log{N} )$



백준(4811) 알약

28 Jul 2021 Algorithm Python Online judge Dynamic programming Baekjoon



문제


백준 알약 문제(4811)

문제를 요약하자면 먼저 병에는 N개의 알약이 존재한다. 하루에 반알씩 약을 복용해야 한다. 한 알 전체를 꺼낼 경우 반으로 쪼갠 후 나머지 조각을 병에 다시 넣고, 반 조각을 꺼낼 경우 그 약을 먹는다. 이때 한 조각을 꺼낼 경우 W, 반 조각을 꺼낼 경우 H를 종이에 기록한다. 2N일이 지났을 때 가능한 서로 다른 문자열의 개수를 구해야 한다.



코드 (Python 3)


import sys
input=sys.stdin.readline
dp  = [[1]*31 if i == 0 else [0]*31 for i in range (31)]
m = 30

for i in range (1,31):
  for j in range(m):
    if j == 0:
      dp[i][0] = dp[i-1][1]
    else:
      dp[i][j] = dp[i-1][j+1] + dp[i][j-1] 
  m-=1

n = int (input ())
while n != 0:  
  print (dp[n][0])
  n = int (input())



풀이


문제가 재귀호출을 사용하는 문제로 분류되어 있어, 처음에는 단순 재귀호출을 사용하여 문제를 해결하고자 했다. 먼저 $F(N,M)$ 는 알약 N 조각과 2등분 된 알약 M 조각이 있을 때 만들어지는 서로 다른 문자열의 개수라고 하자. 만약 알약 통에 각각 N,M 개의 알약이 존재한다면, 등분 되지 않은 알약을 꺼내는 경우와 조각 난 알약을 꺼내는 경우로 나눌 수 있다. 따라서 아래와 같은 점화식을 세울 수 있다.

\[F(N,M) = F(N-1,M+1) + F(N,M-1)\]


처음에는 위 식에 따라 몇가지 조건식을 넣고 recursion 코드를 구현했더니, 중복되는 계산이 많아서 주어진 시간 안에 문제를 풀 수 없을 것 같았다. 따라서 DP를 사용했고 그 중에 Bottom-up 방식으로 다시 문제를 풀게 되었다.


\[F(N,M) = \begin{cases} F(N-1,M+1), & \text{if $N > 0$ and $M = 0 $}\\ 1, & \text{if $N = 0$ and $M > 0$} \\ F(N-1,M+1) + F(N,M-1), & \text{if $N > 0$ and $M > 0$} \end{cases}\]


위 식에 따라 2 X 2 배열을 생성하였고, 입력으로 N(30 이하 정수)이 주어질 때 마다 배열에서 해당 답인 DP[N][0] (나눠지지 않은 알약 N) 찾아서 출력하였다.

시간복잡도


테스트케이스 수와 상관 없이 2차원 배열을 생성해야 하기 때문에 시간복잡도는 $O(N^2)$이다.

백준(1157) 단어공부

18 Jul 2021 Algorithm Python Online judge 구현/시뮬레이션 Baekjoon



코드


python3 :

from collections import Counter

def main ():

    inpStr = input().upper()    
    c = Counter(inpStr)
    
    if len(inpStr) == 1:
        print(inpStr)
        return 
    
    # O(n logk)
    res = c.most_common(2)
    
    if res[0][1] == res[1][1]  : 
        print ('?')
    else : 
        print (res[0][0])
    
    return

main()



풀이


Collections 패키지의 most_common() 메소드를 사용하여 풀이하였다. Counter()를 이용해 문자열의 알파벳과 빈도에 관한 정보를 담은 dictionary 자료형을 만든다. most_common() 메소드에 파라미터로 2를 주면 가장 빈도가 높은 두 개의 알파벳과 그에 대응하는 빈도수 정보를 담은 tuple list가 주어진다. 1,2위 단어의 발생 빈도가 같은 경우 가장 많이 사용된 알파벳이 여러개 존재하는 경우이므로 ‘?’를 출력한다. 그렇지 않은 경우 가장 빈도수가 높은 알파벳 대문자를 출력한다. 주의할 점은 문자열이 알파벳 하나로 이루어졌을 경우인데, 이 경우 허용된 인덱스 값을 벗어나므로 별도로 처리해야된다.

시간복잡도


-> most_common(k)의 시간 복잡도가 O(n logk)이다.
